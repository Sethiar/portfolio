Type

  • Gestion de la sécurité et authentification des utilisateurs
  • Filtrage et modération des commentaires afin d'éviter les contenus inappropriés
  • Automatisation des notifications et gestion des emails
  • Architecture backend/frontend adaptée à un site interactif et modulable

Projet Blog du Youtuber Tititechnique

Type

Application web interactive pour la collecte et la gestion de vidéos

Objectif

Récupérer automatiquement les vidéos d’un YouTuber via l’API YouTube, permettre aux utilisateurs de consulter, liker et interagir avec ces vidéos, tout en assurant une mise à jour asynchrone régulière. Mise en place d'un forum nécessitant de s'authentifier afin de poster un commentaire ou y répondre.

Technologies

Frontend et Backend : Flask (Framework RESTful) ; API YouTube pour récupération des vidéos ; Scheduler Python ( APScheduler) pour mise à jour asynchrone

Base de données

MySQL pour stocker les informations sur les vidéos, titres, nombre de commentaires, nombre de vue, les likes et les informations utilisateur

Hébergement

O2switch

Fonctionnalités clés

  • Récupération automatique des vidéos via l’API YouTube
  • Mise à jour asynchrone avec scheduler pour synchroniser les nouvelles vidéos
  • Système de like sur les vidéos par les utilisateurs
  • Affichage des vidéos sur le site avec tri et filtres
  • Demande de chat vidéo via API WhereBy
  • Authentification et gestion des utilisateurs
  • Forum pour les utilisateurs connectés, commentaires et réponses
  • Mailing afind e suivre l'avancement des sujets discutés
  • Interface administrateur pour superviser les vidéos et interactions

Défis techniques / Ce que j’ai appris

  • Intégration de l’API YouTube pour récupérer les vidéos et métadonnées
  • Intégration d’API externe (WhereBy) pour chat vidéo
  • Gestion des mises à jour asynchrones via scheduler pour ne pas bloquer le site
  • Authentification des utilisateurs afin de poster un message.
  • Implémentation d’un système de likes et suivi des interactions utilisateurs
  • Optimisation des performances pour l’affichage de plusieurs vidéos et métadonnées
  • Architecture backend/frontend robuste pour gérer la synchronisation et les actions utilisateurs

Projet Domotique : Fibaro → IPX → Flask

Type

Application domotique interactive / Contrôle d’objets connectés

Objectif

Contrôler des appareils domotiques Fibaro via IPX et un serveur Flask hébergé sur Raspberry Pi, avec changement dynamique de l’état des icônes sur l'interface utilisateur de la FibaroHC3.

Technologies

Python (Flask) pour le serveur backend ; IPX800 pour la gestion des scénarios ; API et HTTP push de Fibaro ; Raspberry Pi comme serveur local ; Pas de frontend car interface ipx et Fibaro

Base de données

Fichier JSON stockant l’état des appareils et logs

Hébergement

Raspberry Pi local (serveur Flask)

Fonctionnalités clés

  • Création et gestion de scénarios sur IPX pour déclencher des actions
  • Push d’événements IPX vers le serveur Flask
  • Flask reçoit la requête et envoie la commande correspondante à la Fibaro
  • Changement dynamique de l’état des icônes sur l’interface utilisateur
  • Possibilité de contrôler les appareils en temps réel depuis le navigateur
  • Logs et suivi des actions pour vérifier le bon fonctionnement des scénarios

Défis techniques / Ce que j’ai appris

  • Intégration des protocoles Fibaro et IPX pour contrôle domotique
  • Configuration d’un serveur Flask sur Raspberry Pi qui reçoit et traite les requêtes
  • Création de scénarios IPX afin d'automatiser les actions pour déclencher des push
  • Synchronisation temps réel entre état physique des appareils et interface web
  • Gestion de la fiabilité et sécurité des commandes envoyées aux appareils connectés
